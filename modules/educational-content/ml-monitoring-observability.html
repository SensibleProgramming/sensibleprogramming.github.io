01. Overview

In traditional software, monitoring focuses on system health (CPU, memory, uptime). In Machine Learning Operations (MLOps), a healthy server does not guarantee a healthy model.

Once a model is deployed, it interacts with real-world data that is often dynamic and unpredictable. ML Monitoring & Observability is the practice of tracking model behavior in production to ensure it continues to provide business value.

Core Objective Identifying when a model is "stale," tracking its performance, and implementing robust logging to facilitate debugging and retraining.

Detailed Concepts

A. Model Drift

Drift occurs when the statistical properties of the data the model sees in production diverge from the training data.

01

Data Drift

(Feature Drift / Covariate Shift)

  • Definition: Change in input distribution P(X).
  • Context: Model receives data it hasn't "learned" to handle optimally.
  • Detection Methods: KS test, Population Stability Index (PSI), or KL Divergence.
02

Concept Drift

(Relationship Shift)

  • Definition: Change in input-output relationship P(y|X).
  • Example: New fashion trends changing the "meaning" of weather data for sales.
  • Detection Strategy: Monitor error rate over time as ground truth labels arrive.

B. Performance Degradation

Predictive Metrics & Ground Truth Delay
In many production settings, you don't get immediate feedback (e.g., did the user actually default on the loan?).
The Strategy: Monitor proxy metrics (like distribution of predicted probabilities) and track metric decay over time as labels eventually trickle in.

Gold Signal 01

Latency

Time per inference. Critical for real-time fraud detection.

Gold Signal 02

Throughput

Requests per second. Scaling the model with traffic.

Gold Signal 03

Success Rate

Ratio of successful inferences to total requests (4xx/5xx tracking).

C. Logging and Observability

What to Log?

Feature Log (Raw & Transformed)
Inference Log (Output & Confidence)
Model Lineage (Versions & IDs)
Request Metadata (IDs & Tracing)

Implementation Best Practices

A

Asynchronous Processing

Use streams (Kafka/Kinesis) to ensure logging doesn't slow down inference responses.

S

Structured Format

Use JSON for easy indexing and querying in ELK stack or Splunk.

An

Anonymization

Strip or hash PII before logging to maintain GDPR/SOC2 compliance.

Module Summary

ML Monitoring and Observability move the focus from "is the code running?" to "is the model still accurate?". By tracking Model Drift, we catch silent failures before they impact the business. By monitoring Performance Degradation, we ensure a high-quality user experience.
