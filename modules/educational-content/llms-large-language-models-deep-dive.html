1. Overview

This lesson provides an intermediate-level technical deep dive into the architecture and operational constraints of Large Language Models (LLMs). Moving beyond basic definitions, this session explores the underlying Transformer architecture, the mathematical necessity of tokenization, the hardware limitations governing context windows, and the methodologies used to refine raw models into conversational agents.

The goal is to equip learners with a functional understanding of how these models process information and how their structural design influences real-world performance.

Detailed Concept Analysis

How Transformers Work

The Transformer architecture, introduced in the 2017 paper "Attention is All You Need" by Vaswani et al., replaced recurrent neural networks (RNNs) by allowing for parallel processing of data.

The Self-Attention Mechanism

At its core, self-attention allows a model to weigh the importance of different words relative to a specific word. It uses three vectors: Query (Q), Key (K), and Value (V). By calculating the dot product of Q and K, the model determines a focus score.

Encoder vs. Decoder

Encoders (BERT) process entire inputs for classification. Decoders (GPT) generate text one token at a time using "masked self-attention," proving more effective for open-ended synthesis.

Technical Insight

Modern LLMs are primarily decoder-only because they are optimized for Autoregressive Generation. This architecture scales more effectively with massive datasets compared to bi-directional encoders.

Token Limits & Tokenization

LLMs do not "read" text; they process numerical representations of text chunks called tokens. This is the critical bridge between human language and vector space.

Byte Pair Encoding (BPE) Logic
  • The Process: Iteratively merges frequent adjacent pairs into new, sub-word tokens.
  • Token vs. Word: One token ≈ 0.75 words. Rare words or code are broken into multiple units.
  • Spelling Issues: LLMs struggle with reverse-strings because they see "chunks," not individual letters.
  • Efficiency: Better tokenization allows the model to "see" more information within fixed memory.

Context Windows

The context window is the maximum number of tokens a model can consider at one time (including both input and generated output).

O(n²)

Quadratic Complexity: Doubling the window quadruples the memory requirements.

Impact on RAG

Larger windows allow more grounding evidence but can lead to "Lost in the Middle" phenomena where recall dips for central information.

Fine-tuning Basics

A raw "Base Model" is trained on the internet to predict the next token, but it isn't helpful or safe. Fine-tuning transforms it into a specialized tool.

Supervised Fine-Tuning (SFT)
The model is trained on a curated dataset of "Instruction-Response" pairs. This teaches the model the format of a helpful assistant.
RLHF (Human Feedback Alignment)
Humans rank model outputs. These rankings train a Reward Model, which uses Proximal Policy Optimization (PPO) to nudge the LLM toward outputs that align with human preferences (helpfulness, honesty, harmlessness).
PEFT & LoRA
Standard fine-tuning is hardware-intensive. Low-Rank Adaptation (LoRA) freezes original weights and only trains a small, low-rank matrix injected into the layers, allowing high-performance tuning on consumer hardware.

4. Final Summary

Understanding LLMs requires looking past the interface to the Transformer's self-attention engine. We see that tokenization defines the model's vocabulary and efficiency, while the context window represents a hard physical limit dictated by quadratic math. The journey from a base model to a functional tool is achieved through a pipeline of SFT, RLHF, and efficient tuning like LoRA, which aligns raw statistical prediction with human intent.
